  D-01, D-02 결정에 따라 RTK를 Homebrew로 설치하고 `rtk gain` 실행으로 올바른 바이너리임을 검증한다. `cargo install rtk`는 crates.io의 "Rust Type Kit"을 설치하므로 절대 사용 금지 (Pitfall C1).

  정확한 실행 순서:

  1. 설치 전 사전 확인:
     ```bash
     brew --version    # 5.0.7+ 기대
     brew info rtk     # formula v0.37.2+ 확인
     which rtk         # 미설치면 "rtk not found" 기대
     ```

  2. RTK 설치:
     ```bash
     brew install rtk
     ```

  3. 설치 직후 바이너리 경로 확인:
     ```bash
     which rtk
     # Expected: /home/linuxbrew/.linuxbrew/bin/rtk
     ```

  4. 올바른 RTK 바이너리 검증 (Pitfall C1 방어):
     ```bash
     rtk gain
     # Expected: 토큰 절감 통계 출력 (예: "Token savings: ..." 형식)
     # FAIL signal: "unrecognized command", help text 출력, 또는 Rust Type Kit 스타일 출력
     ```

  5. 버전 기록:
     ```bash
     rtk --version
     # Expected: v0.37.2 이상
     ```

  주의:
  - D-08: 프로젝트 파일 수정 없음. RTK는 개인 도구.
  - D-07: PATH 조작 불필요. Linuxbrew bin은 이미 PATH 등록됨.
  - `rtk gain`이 네트워크 호출을 요구한다면 offline mode로도 성공하는지 확인. 실패 시 stderr 전체를 SUMMARY에 기록.

  `rtk gain` 검증 실패 시 즉시 중단:
  ```bash
  brew uninstall rtk
  # 재시도 or 사용자 보고
  ```

  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| brew formula → local binary | Homebrew가 제3자 formula를 fetch하여 로컬 바이너리 설치 — supply chain 경계 |
| RTK 바이너리 → shell 실행 권한 | 설치된 바이너리가 현재 사용자 권한으로 실행됨 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Spoofing | brew formula "rtk" (동명 패키지 혼동) | mitigate | `rtk gain` 실행으로 올바른 rtk-ai/rtk 바이너리임을 즉시 검증. Pitfall C1 대응. crates.io rtk(Rust Type Kit) 설치 방지. |
| T-09-02 | Tampering | /home/linuxbrew/.linuxbrew/bin/rtk (supply chain) | accept | Homebrew 공식 formula 신뢰. 별도 서명 검증 인프라 없음 — 개인 개발 환경 범위. brew formula가 공식 rtk-ai 소스임은 `brew info rtk`의 homepage 필드로 교차 확인. |
| T-09-03 | Information Disclosure | `rtk gain` 네트워크 호출 시 텔레메트리 | mitigate | Plan 02에서 `RTK_TELEMETRY_DISABLED=1` 설정 후 Plan 02의 재검증 단계에서 gain 재실행. 이 Task에서는 초기 확인만 수행. |
| T-09-04 | Elevation of Privilege | brew install 시 권한 상승 | accept | Linuxbrew는 user-scope 설치 (루트 권한 불필요). /home/linuxbrew가 현재 사용자 소유. |
</threat_model>
