### 현재시각 — RTK hook과 GSD hook 충돌 없음 분석

- settings.json PreToolUse 구조: [0~2] Write|Edit matcher(GSD guard), [3] Bash(gsd-validate-commit.sh), [4] Bash(rtk hook claude)
- GSD Write/Edit guard 3개는 RTK(Bash only)와 matcher 레벨에서 완전 분리 — 겹치지 않음
- gsd-validate-commit.sh는 opt-in 구조: config.json에 hooks.community: true 없으면 즉시 exit 0 — 현재 비활성
- 실행 순서: [3] exit 0 → [4] rtk rewrite. [3]이 block(exit 2)하면 [4]는 실행 안 됨 — 동시 수정 불가 구조
- hooks.community 활성화 시에도: git commit 메시지 위반이면 [3]이 차단, RTK는 실행 안 됨 → 충돌 없음
